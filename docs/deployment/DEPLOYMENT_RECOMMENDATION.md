# Deployment Recommendation

> Verdict and configuration for hosting KomunaID.

## Verdict

**Vercel is not recommended** as the production host for KomunaID. The application is a stateful Laravel monolith that requires:

- Writable filesystem (logs, cache, session files, uploaded files).
- Long-running PHP process model (PHP-FPM + Nginx) for stable sessions and queue workers.
- Managed MySQL with persistent connections.
- A cron / scheduler for `php artisan schedule:run`.
- Ability to run `php artisan migrate --force` on deploy.

Vercel's serverless functions are stateless, short-lived, and have a read-only filesystem at runtime. None of the above can be satisfied cleanly.

## Recommended Targets (in order of fit)

1. **VPS + Laravel Forge** — $6–25/mo for the server + managed MySQL add-on. Best DX for teams.
2. **VPS + Ploi** — similar to Forge, different UI/price.
3. **VPS + RunCloud** — similar.
4. **Managed cPanel / business hosting** — only if zero-dev-ops is a hard requirement. Limits queue/cron.

## Required Environment Variables (production)

```
APP_NAME=KomunaID
APP_ENV=production
APP_DEBUG=false
APP_KEY=<generated by php artisan key:generate>
APP_URL=https://<your-domain>

# Database (managed MySQL)
DB_CONNECTION=mysql
DB_HOST=<db-host>
DB_PORT=3306
DB_DATABASE=komunaid
DB_USERNAME=<db-user>
DB_PASSWORD=<db-password>

# Stateful drivers
SESSION_DRIVER=database
CACHE_STORE=database
QUEUE_CONNECTION=sync

# Logging (Forge default)
LOG_CHANNEL=stack
LOG_LEVEL=info

# Mail
MAIL_MAILER=smtp
MAIL_HOST=<smtp-host>
MAIL_PORT=587
MAIL_USERNAME=<smtp-user>
MAIL_PASSWORD=<smtp-password>
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=no-reply@<your-domain>
MAIL_FROM_NAME="${APP_NAME}"

# File storage
FILESYSTEM_DISK=public
PUBLIC_DISK_DRIVER=local
```

> Real values are placeholders. Do not commit actual secrets.

## Pre-Deploy Checklist

```
composer install --no-dev --optimize-autoloader
npm ci
npm run build
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan storage:link
```

Verify `public/build/manifest.json` exists after `npm run build`. If it does not, assets are 404 in production.

## If Vercel is Still Required (fallback)

These are the minimum changes to make the app *boot* on Vercel — but the platform remains fragile for a full Laravel app:

- Set `SESSION_DRIVER=cookie` and `CACHE_STORE=array`.
- Set `LOG_CHANNEL=stderr`.
- Move uploads to S3 / Cloudflare R2 (`FILESYSTEM_DISK=s3` + R2 credentials).
- Pre-build assets and commit `public/build/` (Vercel cannot run `npm run build` with the current `vercel.json`).
- Use external MySQL (PlanetScale / Aiven / Railway) with SSL env vars.
- Accept that cron / queue / scheduler are unavailable.

This mode is not recommended for production.

## Local Development

- XAMPP (MySQL) on `127.0.0.1:3306`, user `root`, no password.
- `php artisan serve` on `http://127.0.0.1:8000`.
- `npm run dev` for Vite HMR.
- `.env` and `.env.example` are local; `.env.testing` for tests; `.env.local` and `.env_local` are duplicates and should be removed in cleanup.
